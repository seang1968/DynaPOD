In PowerShell, when listing mapped network drives, the path can sometimes be truncated in certain views. You can use the Get-WmiObject command to retrieve the full UNC path of the mapped network drives without truncation. Here's how you can do it:

powershell Get-WmiObject -Query "SELECT * FROM Win32_NetworkConnection" | Select-Object LocalName, RemoteName

This command will print out both the local drive letter (LocalName) and the full UNC path (RemoteName) without chopping the path.

If you're specifically looking for a particular drive, you can filter the output as well:

powershell Get-WmiObject -Query "SELECT * FROM Win32_NetworkConnection WHERE LocalName='Z:'" | Select-Object LocalName, RemoteName

This will return only the details for the mapped drive Z:. You can replace 'Z:' with the letter of your mapped drive.
